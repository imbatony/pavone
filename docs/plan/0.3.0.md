# PAVOne v0.3.0 开发计划 — 工程质量提升

## 📋 版本概述

本版本聚焦于**工程质量与开源规范化**，不新增业务功能，专注于将项目从"能用"提升到"优秀开源项目"标准。

参照标杆项目：ruff、httpie、click。

**主要目标**:

1. 消除配置文件冲突与冗余，建立单一真相源
2. 建立安全默认值，修复 SSL 验证问题
3. 建立完整的异常层级体系
4. 统一日志/输出规范，消除 print() 滥用
5. 补齐开源项目标准文件与开发工具链

## 阶段二：安全性与异常体系（优先级：严重）

### 2.1 修复 SSL 验证默认值

**背景**: `pavone/utils/http_utils.py` 中 `verify_ssl` 默认为 `False`，违反安全默认原则。

**任务**:

- [ ] 将 `pavone/utils/http_utils.py` 中 `fetch()` 的 `verify_ssl` 默认值改为 `True`
- [ ] 审查所有调用 `http_utils.fetch()` 的地方，确认不会因此破坏功能
- [ ] 对确实需要跳过验证的场景（如自签名证书），在调用处显式传入 `verify_ssl=False` 并添加注释说明原因
- [ ] Jellyfin 客户端禁用 SSL 验证时增加日志警告

**验收标准**: 默认安全，仅在显式声明时才跳过验证。

---

### 2.2 建立全局异常层级体系

**背景**: 全项目约 89-170 处使用 `except Exception`，仅 `jellyfin/exceptions.py` 有自定义异常。

**任务**:

- [ ] 创建 `pavone/core/exceptions.py`，定义异常层级：

```python
class PavoneError(Exception):
    """PAVOne 基础异常"""

class DownloadError(PavoneError):
    """下载过程中的错误"""

class ExtractError(PavoneError):
    """提取（URL/元数据）过程中的错误"""

class PluginError(PavoneError):
    """插件系统错误"""

class ConfigError(PavoneError):
    """配置相关错误"""

class MetadataError(PavoneError):
    """元数据处理错误"""

class NetworkError(PavoneError):
    """网络请求错误"""
```

- [ ] 让 `jellyfin/exceptions.py` 中的 `JellyfinException` 继承自 `PavoneError`
- [ ] **逐步替换** `except Exception` 为具体异常类型（优先处理以下文件）：
  - `pavone/core/downloader/` — 使用 `DownloadError`, `NetworkError`
  - `pavone/plugins/base.py` — 使用 `PluginError`, `NetworkError`
  - `pavone/cli/commands/` — 在最外层捕获 `PavoneError`
  - `pavone/config/manager.py` — 使用 `ConfigError`
- [ ] CLI 入口点统一捕获 `PavoneError`，输出友好错误信息

**验收标准**: 核心模块不再使用裸 `except Exception`，CLI 层仅在最外层做兜底捕获。

## 阶段三：代码质量规范化（优先级：重要）

### 3.1 消除 print() 滥用

**背景**: 源码中 43+ 处 `print()` 调用，绕过日志系统。

**分类处理原则**:
- **用户交互输出**（搜索结果、进度提示）→ 改用 `click.echo()`
- **错误/警告信息**（config/manager.py 中的 ERROR/WARNING）→ 改用 `logger.error()` / `logger.warning()`
- **调试信息** → 改用 `logger.debug()`

**任务**:

- [ ] `pavone/config/manager.py` — 将 `print(f"ERROR: ...")` 改为 `logger.error()`
- [ ] `pavone/cli/commands/search.py` — 用户输出改用 `click.echo()`
- [ ] `pavone/cli/commands/jellyfin.py` — 同上
- [ ] `pavone/cli/commands/metadata.py` — 同上
- [ ] `pavone/cli/commands/enrich_helper.py` — 同上
- [ ] `pavone/jellyfin/download_helper.py` — 同上
- [ ] `pavone/manager/progress.py` — 用户输出改用 `click.echo()`
- [ ] `pavone/manager/execution.py` — 交互提示改用 `click.echo()`

**验收标准**: `grep -r "print(" pavone/ --include="*.py"` 返回 0 结果（或仅剩有合理理由的极少数）。

## 阶段五：CI/CD 加固（优先级：重要）

### 5.1 安全扫描不再静默忽略

**背景**: CI 中 `safety check || true` 和 `bandit || true` 使安全问题不会阻断 PR。

**任务**:

- [ ] 移除 `safety check` 后的 `|| true`（或改为 `continue-on-error: false`）
- [ ] 移除 `bandit` 后的 `|| true`
- [ ] 如有已知误报，使用 `# nosec` 注释或 `.bandit` 排除文件处理

**验收标准**: 安全扫描失败时 CI 会标红。

---

### 5.2 依赖版本添加上界约束

**背景**: 所有依赖使用 `>=` 无上界，大版本升级可能破坏功能。

**任务**:

- [ ] 为主要依赖添加上界约束：

```toml
dependencies = [
    "requests>=2.31.0,<3",
    "beautifulsoup4>=4.12.0,<5",
    "click>=8.1.0,<9",
    "rich>=13.0.0,<15",
    "pydantic>=2.12.4,<3",
    "lxml>=4.9.0,<6",
    "pillow>=10.0.0,<12",
    # ...
]
```

- [ ] 运行 `uv lock` 验证依赖解析无冲突
- [ ] 运行测试确认无破坏

**验收标准**: 所有依赖有合理的上界约束，`uv lock && uv run pytest` 通过。

1. **向后兼容**: 本版本不引入破坏性 API 变更。删除 setup.py 不影响 `uv` / `pip` 安装。

3. **异常替换策略**: 采用渐进式替换，不要求一次性替换所有 `except Exception`。优先处理 CLI 层和核心模块。
